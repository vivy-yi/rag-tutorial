
## 💡 学习建议

### 理论学习

1. **先理解概念**，不要急于写代码
2. **做好笔记**，记录关键要点
3. **画图理解**，可视化RAG流程
4. **对比学习**，理解不同方案的优劣

### 实践练习

1. **跟随代码**，一步步运行示例
2. **修改参数**，观察效果变化
3. **提出问题**，尝试自己解决
4. **完成项目**，巩固所学知识

### 常见问题

**Q1: 学习曲线陡峭怎么办？**

A: 这是正常的！建议：
- 每天学习1-2章，不要贪多
- 重点理解第3章的实现
- 多运行代码，在实践中理解

**Q2: 代码运行出错？**

A: 常见解决方案：
- 检查依赖版本：`pip list`
- 确认API密钥已设置
- 查看错误信息，逐步排查
- 参考示例代码对比

**Q3: 如何检验学习效果？**

A: 通过以下方式：
- 完成每章的练习题
- 运行示例代码并理解输出
- 完成第5章的实战项目
- 达到评估指标基准（Hit Rate > 0.6）
