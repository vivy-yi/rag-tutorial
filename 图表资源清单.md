# 图表使用规范

## 图片引用格式

在Markdown中引用图片的标准格式：

```markdown
![图片说明](relative/path/to/image.png)
*图X-Y：图片的详细说明文字*
```

## 图片处理建议

1. **大图处理**：重要架构图使用完整尺寸
2. **缩略图**：对比图可适当缩小
3. **图注**：每张图都要有清晰的说明
4. **交叉引用**：在正文中引用图片编号

## 图片组织建议

### 按模块组织目录结构

```
RAG完整教程/
├── images/
│   ├── module1-basic/       # 模块1图片
│   ├── module2-optimization/ # 模块2图片
│   ├── module3-advanced/     # 模块3图片
│   ├── module4-production/   # 模块4图片
│   └── logos/                # Logo和品牌资源
```

### 图片命名规范

- 使用描述性文件名
- 使用连字符分隔单词
- 小写字母
- 例如：`agentic-rag-architecture.svg`
